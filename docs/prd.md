# PRD：Agent Detector + Humanizer（v1 产品视角）

## 1. 产品一句话
面向内容创作者与小团队的“可信输出系统”：一键识别 AI 迹象与风险、并将内容改写为更真实、更具个人风格的版本，同时支持逐步训练个人化 Agent。

## 2. 目标与价值主张
- 目标：让用户在“风险可控、风格一致、效率可衡量”的前提下完成高质量内容输出。
- 价值主张：
  - 可解释的检测：不仅给分数，还告诉你哪里“像 AI”以及为什么。
  - 目标导向的人类化：不是泛化重写，而是面向“读者相信感与品牌一致性”的改写。
  - 个人化 Agent：把用户的写作习惯沉淀成资产，减少反复改写的成本。

## 3. 适用用户与场景
- 个人创作者：需要保持“人设与语气稳定”，同时避免内容被标记为 AI。
- 品牌/市场团队：需要统一品牌语调，降低团队协作中风格漂移。
- 教育/培训与职业材料：需要严肃、可信、正式且自然的表达。

## 4. 核心问题（用户痛点）
- “我写得快，但看起来像 AI。”
- “我写得像人，但不稳定、成本高。”
- “团队写作风格不一致，改来改去。”
- “检测工具只给分数，不告诉我怎么改。”

## 5. v1 目标范围
- 模块 1：AI 内容 Detector
  - 可解释性风险提示（如结构化痕迹、语气不自然、模板化语句、重复节奏）。
- 模块 2：Humanizer（人类化改写）
  - 以检测结果为输入，提供有目标的改写（更口语、更专业、更个人化）。
- 模块 3：个人化 Agent 训练（轻量版本）
  - 以用户反馈与偏好为核心的风格逐步贴合。

## 6. 非范围（v1 不做）
- 登录与支付
- 多人协作与角色权限
- 公开 API 与插件生态

## 7. 关键差异点（更“出彩”的命中需求）
- “看得懂的风险解释”：把“抽象分数”变成“具体可改的点”。
- “读者可信感作为第一目标”：不是纯去 AI 化，而是更像“这个人写的”。
- “风格记忆资产化”：用户的风格不是一次性设置，而是持续可复用的资产。
- “改写可控而非随机”：明确目标风格与语气，不是无方向的改写。

## 8. 成功指标（产品视角）
- 检测到改写的转化率
- 改写后的“可用率”（用户无需再次大幅修改的比例）
- 用户对“解释性提示”的采纳率
- 个人化 Agent 的复用率与留存

## 9. 风险与假设
- 假设：用户愿意为了“可信与风格一致”接受轻量训练流程。
- 风险：若解释性提示过于抽象，用户会回到“只看分数”。
- 风险：改写如果缺乏风格控制，会被认为“只是换句子”。

## 10. v1 版本边界说明
本版本以“产品概念与需求描述”为主，不涉及任何技术细节或实现方式。
