# 用户 Happy Map（正向旅程）

## 目标用户
以“需要稳定高质量输出、且在意风格与可信感”的内容生产者为主，包括：
- 个人创作者
- 品牌与市场团队
- 教育/职业材料用户

---

## Happy Map 结构
每一阶段描述：动机 / 行为 / 触点 / 情绪变化 / 关键收获

---

## 阶段 1：触发与发现
- 动机：担心内容被认为“太像 AI”，或风格不够统一
- 行为：寻找检测工具或改写方案
- 触点：Landing 页面、案例对比、真实用户反馈
- 情绪变化：焦虑 → 看到解决方向后放心
- 关键收获：明确这是“检测 + 改写 + 个人化”的一体方案

---

## 阶段 2：首次检测（价值被看见）
- 动机：快速确认风险点
- 行为：粘贴文本 → 一键检测
- 触点：检测分数 + 风险解释标签
- 情绪变化：不确定 → 可理解
- 关键收获：知道“问题在哪里”，不再依赖模糊分数

---

## 阶段 3：一键人类化（价值被验证）
- 动机：希望马上得到可用版本
- 行为：选择目标风格 → 生成改写
- 触点：改写结果 + 对比展示 + 修复说明
- 情绪变化：焦虑 → 轻松
- 关键收获：内容可直接使用，且更“像自己”

---

## 阶段 4：偏好反馈（从一次性到长期）
- 动机：让输出越来越符合个人风格
- 行为：点赞/不喜欢、标注偏好
- 触点：反馈入口 + 风格选项
- 情绪变化：被动使用 → 主动塑形
- 关键收获：风格开始沉淀为“个人资产”

---

## 阶段 5：复用与依赖
- 动机：更快产出、更稳定风格
- 行为：持续使用个人化 Agent
- 触点：个人化 Agent 的命名与选择
- 情绪变化：试用 → 依赖
- 关键收获：输出稳定、效率提升，产品成为写作工作流的一部分

---

## 旅程中的“出彩时刻”
- 检测解释非常直观：用户立刻知道“为什么像 AI”
- 改写对比清晰：用户理解“具体改了什么”
- 风格逐步贴合：用户感到“这是我写的”，而非“被改写的”

---

## v1 边界说明
本 Happy Map 聚焦“产品体验与用户心理”，不涉及技术实现、登录或支付流程。
